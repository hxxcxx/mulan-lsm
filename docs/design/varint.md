# Varint — 整数压缩

**原理**：每字节低 7 位存数据，最高位标记"是否还有后续字节"（1=还有，0=结束）。

| 数值 | varint 占用 | 固定 u32 占用 |
|------|------------|--------------|
| 0~127 | 1 字节 | 4 字节 |
| 128~16383 | 2 字节 | 4 字节 |
| ... | ... | 4 字节 |
| u32::MAX | 5 字节 | 4 字节 |

**编码**：`while value >= 0x80 { push(low7 | 0x80); value >>= 7 }` → push 最后 7 位

**解码**：逐字节取低 7 位拼接到 result，最高位为 0 时停止

**在项目中的使用**：
- Block entry header 的三个整数（shared / non_shared / value_len）—— 大部分 ≤ 127，1 字节
- BlockHandle 的 offset / size —— 1~3 字节
- WAL entry 的 key_len / val_len —— 1 字节

**定长 vs 变长的混用（WAL encode_entry）**

| 字段 | 编码 | 为什么定长/变长 |
|------|------|----------------|
| vtype | 1 字节定长 | 只有 2 种值 |
| seq | u64 小端定长 8 字节 | 单调递增到很大，没有压缩意义 |
| key_len | varint | 通常很小（几十字节），节省空间 |
| val_len | varint | 同 key_len |

定长：解析快、位置可预知。变长：省空间。根据场景混用。
